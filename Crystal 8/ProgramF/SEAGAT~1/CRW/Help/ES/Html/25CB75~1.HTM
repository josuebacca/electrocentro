Formatos comunes de bases de datos

Aunque Seagate Crystal Reports use el mismo sistema de tres capas para obtener datos de todos los formatos de archivos de bases de datos de acceso directo, cada formato requiere de un conjunto de DLL diferente. Sin embargo, algunos formatos expanden la estructura básica de tres capas.

Las secciones siguientes cubren los sistemas usados por Seagate Crystal Reports para acceder a algunos de los formatos de bases de datos más usados.

dBASE, FoxPro, Clipper

Seagate Crystal Reports ha sido diseñado para abrir datos dBASE simple y directamente mediante el motor xBase (en P2bxbse.dll). FoxPro y Clipper son formatos de bases de datos compatibles con dBASE y Seagate Crystal Reports usa la misma DLL para acceder a los archivos creados por cualquiera de estas tres aplicaciones DBMS.

Nota:    La capa de traducción P2bxbse.dll es compatible con archivos FoxPro hasta la versión 2.x. Vea Visual FoxPro para versiones posteriores a la 2.5.

El archivo P2bxbse.dll maneja toda la traducción entre Seagate Crystal Reports y los archivos dBASE, FoxPro o Clipper. Cada archivo de base de datos sólo contiene una sola tabla de base de datos, pero no existe un límite de archivos accesibles por un informe.

Nota:    También es posible acceder a los datos dBASE mediante el motor de base de datos Borland (BDE) usando el archivo de traducción P2bbde.dll. Para ver cómo se comunica BDE con los datos de la base de datos, vea Paradox. Sin embargo, el BDE no es compatible con datos FoxPro ni datos Clipper.
