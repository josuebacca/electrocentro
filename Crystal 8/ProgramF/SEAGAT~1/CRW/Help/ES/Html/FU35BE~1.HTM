ExchGetId (dirección)

Sintaxis básica y Crystal.

Argumento

dirección es la dirección del emisor/receptor (tipo de datos de cadena).

Devuelve

Una cadena

Acción

La función ExchGetID (dirección) empieza por determinar si la dirección está en formato X500 o X400. Una vez que esto ha sido solucionado, entonces la función puede determinar la identificación.

  • Si el campo está en formato X500, la función extraerá la última instancia del código "CN=" code (no sensible a mayúsculas/minúsculas).
  • Si el campo está en formato X400, extraerá las Ids de SMTP o MS.

Nota:    Los campos deben seguir los estándares de tipo de dirección para las funciones para operarlos.

Ejemplo(s)

Los siguientes ejemplos son aplicables para la sintaxis Crystal:

Campo = /o=Seagate Software/ou=CRYSTALNT
/cn=Configuration/cn=Servers/cn=ESPRESSO/cn=Fredt
ExchGetID({Campo})

Devuelve "Fredt".

Campo = c=US; a= ;p=Microsoft; o=apps-wga;
dda:smtp=James@crystal.com
ExchGetID({Campo})

Devuelve "James@crystal.com".

Campo = c=US; a= ;p=Microsoft; o=apps-wga; dda:ms=com/
crystal/James
ExchGetID({Campo})

Devuelve "com/crystal/James".
