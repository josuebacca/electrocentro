BeforeReadingRecords

Sintaxis básica y Crystal.

Devuelve

Nada

Acción

Especifica que la fórmula va a ser evaluada antes que los registros de la base de datos sean leídos.

Uso(s) típico(s)

Las fórmulas son evaluadas normalmente en las siguientes ocasiones  :

  • Si ninguna base de datos o ningún campo de grupo está incluido en la fórmula, la fórmula es evaluada antes de que el programa lea los registros de la base de datos.
  • Si una base de datos está incluida en la fórmula, la fórmula es evaluada mientras el programa lee los registros de la base de datos.
  • Si un campo de grupo, campo de número de página, subtotal, etc., está incluido en la fórmula, la fórmula es evaluada después de que los registros de la base de datos son leídos y mientras los datos de los registros son impresos en el informe.

BeforeReadingRecords obliga a la fórmula a ser evaluada antes de que el programa lea los registros de la base de datos. Cuando esta función es utilizada en una fórmula, el Verificador de Fórmula devuelve un mensaje de error si intenta incluir elementos en la fórmula (campos, grupos de base de datos, etc.) que deben de ser evaluados después (mientras se leen o se imprimen los registros).

Ejemplo(s)

El siguiente ejemplo es aplicable a la sintaxis Crystal:

BeforeReadingRecords;
ToNumber ("12345")

Obliga a la fórmula (la cual no contiene campos o grupos de base de datos), a ser evaluada antes de leer los registros.

Nota:    Si trata de incluir un campo o un grupo de base de datos en esta fórmula, usted obtendrá un mensaje de error.
