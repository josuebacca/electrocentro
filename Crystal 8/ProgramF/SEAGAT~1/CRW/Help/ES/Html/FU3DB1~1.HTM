UBound (matriz)

Sintaxis básica y Crystal.

Argumento

matriz es un valor, expresión o variable de matriz.

Devuelve

Valor Número.

Acción

UBound devuelve un Número que contiene el subíndice más grande disponible para una matriz dada.

Uso típico

Se usa normalmente para obtener el tamaño de una matriz antes de usar un mecanismo de control de bucle para manipular sistemáticamente los elementos de la matriz.

Ejemplo(s)

Los siguientes ejemplos son aplicables a las sintaxis básica y Crystal:

Sintaxis básica Rem
Dim matrizSimple (10) Como Número
formula = UBound(matrizSimple)

//Sintaxis Crystal
Local NumberVar Array matrizSimple;
Redim matrizSimple[10];
UBound(matrizSimple)

Devuelve 10.

Sintaxis básica Rem
Dim matrizFecha () Como Fecha
matrizFecha = Array (CDate(#12/25/1998#), CDate(#12/24/1999#))
formula = UBound(matrizFecha)

Sintaxis Crystal
Local DateVar Array matrizFecha;
matrizFecha = Array (CDate(#12/25/1998#), CDate(#12/24/1999#))
UBound (matrizFecha)

Devuelve 2.

Comentario(s)
  • Esta función está diseñada para trabajar exactamente como la función de Visual Basic del mismo nombre.

    Nota:    A diferencia de Visual Basic, sólo se admite una matriz dimensional en Seagate Crystal Reports. Además, los subíndices de matriz empiezan en 1 en Seagate Crystal Reports y no en 0, como en el valor predeterminado de Visual Basic.

  • UBound devuelve 1 para una variable de matriz no inicializada; sin embargo, indexar una variable de matriz no inicializada causa un error. La razón de este comportamiento es que de momento Seagate Crystal Reports no admite matrices con cero elementos. Para superar este problema, siempre inicialice sus variables de matriz. Por ejemplo, inicialice variables de matrices globales en una fórmula localizada en el encabezamiento del informe y variables de matrices locales dentro de la propia fórmula.
