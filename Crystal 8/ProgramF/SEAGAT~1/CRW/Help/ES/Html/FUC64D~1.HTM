Cadenas de formato

Las funciones ToText y CStr puede utilizar cadenas de formato para controlar como aparecerán los números fraccionarios, los valores monetarios, de fecha, hora, o de fecha/hora que pasan al argumento x cuando son convertidos a una cadena. Esta sección describe como crear cadenas de formato para los varios tipos de datos.

Cadenas de formato para números fraccionarios y valores monetarios 

Los siguientes caracteres son utilizados para crear cadenas de formato para números fraccionarios y valores monetarios  :

Carácter Descripción Comentario(s) /Ejemplos

#

# representa dígitos numéricos o espacios. Cuando el número original es convertido, los caracteres # son ignorados a menos que el número de caracteres # en la cadena de formato se extienda más allá del número de dígitos significativos en el valor original. Después, por cada # adicional que aparezca, un espacio es agregado a la cadena.

Si su valor original es 125.34:

  • la cadena de formato "#" produce la cadena "125" (el número de dígitos significativos excede el número de caracteres #)
  • la cadena formato "###.##" produce la cadena "125.34" (el número de dígitos significativos es igual al número de caracteres #)
  • el formato de cadena "####.###" produce la cadena "125.34" (el número de dígitos significativos es menor que el número de caracteres # ).

0

0 representa dígitos numéricos o ceros (0). Cuando el número original es convertido, los caracteres 0 son ignorados a menos que el número de caracteres 0 en la cadena de formato se extienda más allá del número de dígitos significativos en el valor original. Después, por cada 0 adicional que aparezca, un cero es agregado a la cadena.

Si su valor original es 125.34:

  • la cadena de formato "0" produce la cadena "125" (el número de dígitos significativos excede al número de caracteres 0)
  • la cadena de formato "000.00" produce la cadena "125.34" (el número de dígitos significativos es igual al número de caracteres 0)
  • la cadena de formato "0000.000" produce la cadena "0125.340" (el número de dígitos significativos es menor que el número de caracteres 0).

los caracteres # y 0 pueden ser combinados en cadenas de formato. Si el valor original es 125.34:

  •     la cadena de formato "####.0000" produce la cadena "125.3400".
  • Una coma (,) indica dónde debe de aparecer un separador de miles. El carácter real que aparece como un separador de miles en la cadena resultante puede ser cambiado. Refiérase a los argumentos para convertir valores decimales y monetarios con la función ToText en la sección Argumentos anterior.
  • Un punto decimal (.) indica cuándo un separador decimal debe aparecer. El carácter real que aparece como separador decimal en la cadena resultante puede ser cambiado.

coma (,)

Muestra dónde debiera aparecer un indicador de millar.

En la cadena resultante, se puede cambiar el carácter actual que aparece como separador de millares.


punto decimal (.)

Indica dónde debiera aparecer un separador decimal.

En la cadena resultante, se puede cambiar el carácter actual que aparece como separador de decimales.
