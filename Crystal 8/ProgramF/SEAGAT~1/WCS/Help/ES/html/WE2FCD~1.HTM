Configuración de Lotus Domino con el Web Component Server

Si va a usar un servidor Lotus Domino, siga el procedimiento que se describe a continuación antes de acceder a informes a través del web.

  1. Copie el archivo wcscgi.exe en el directorio cgi-bin de Lotus Domino.

    Esto proporciona al servidor web Domino acceso a los archivos de programa del servidor web Crystal Reports a través de su directorio CGI.

  2. Cree un directorio virtual para Seagate Crystal Reports en Lotus Domino.

    Para hacerlo, use la base de datos de configuración de Domino. Si no tiene una base de datos de configuración de Domino, cree una denominada domcfg.nsf en el directorio predeterminado de datos del servidor Domino usando la plantilla de configuración de Domino (domcfg.ntf).

  3. Asigne las URL al informe.

    La forma de hacerlo variará dependiendo de la versión de Domino que esté usando. Para más información, consulte la ayuda en línea de Domino.

      • Asignación de directorios en Lotus Domino versión 4.x

    Mediante la base de datos de configuración de Domino, siga la ruta Crear -> Asignación de URL -> Directorio y asigne los siguientes cuatro directorios:
    Ruta de URL Directorio

    1

    /scrreports/*

    c:\Progra~1\Seagat~1\Crysta~1\Reports\*

    2

    /scrsamples/*

    c:\Progra~1\Seagat~1\Crysta~1\samples\ code\web\*

    3

    /viewer/*

    c:\Progra~1\Seagat~1\Viewers\*

    4

    /reports

    d:\notes\data\report

    Las primeras dos asignaciones son opcionales. Le permiten ver muestras incluidas con el producto.

    La tercera asignación es necesaria, ya que éste es el directorio donde se almacenan los clientes del visor. Cuando un explorador solicita un informe, el Web Component Server interpreta el explorador y carga automáticamente el visor apropiado (Java, ActiveX, HTML o HTML con marcos).

    La cuarta asignación es necesaria. Ésta determina el directorio actual en el servidor cada vez que se recibe una solicitud de informe. El nombre del directorio (en este caso /reports) puede ser cualquiera que decida utilizar en el directorio \notes\data. Se activa mediante el redireccionamiento de URL, que se muestra a continuación.

    En la asignación de directorios, puede especificar la dirección IP a la cual se aplica la asignación. Esto es necesario solamente si va a usar varios servidores virtuales y desea restringir el acceso a sólo un informe. Todas las asignaciones de directorios deben establecerse también con acceso de lectura, no de ejecución. Se recomienda introducir información de referencia en el campo de comentarios, pero esto es opcional.

      • Asignación de directorios en Lotus Domino versión 5.x

    Abra el administrador de Domino. Haga clic en la ficha de configuración y abra el servidor ->Vista de documentos actuales del servidor A continuación, seleccione el botón de web. Cree un redireccionamiento de asignación de directorios URL.

    Lectura

    Opcional

    2

    /scrsamples/*

    c:\Archivos de programa\Seagate Software\Crystal Reports\Samples\*

    Lectura

    Opcional

    3

    /viewer/*

    C:\Archivos de programa\Seagate Software\Viewers\*

    Lectura

    Opcional

    4

    d:\notes\data\report

    Lectura

    Opcional

  4. Reinicie el servidor Lotus Domino.
  5. Asegúrese de que los servicios NT se hayan iniciado.
      • Seleccione Servicios en el Panel de control.

        Aparecerá el cuadro de diálogo Servicios.

      • Compruebe que los servicios Seagate Page Server y Seagate Web Component Server se hayan iniciado. Si no se están ejecutando, haga clic en Inicio.

  6. Para acceder a los informes de muestra, introduzca la URL en su explorador web.

    http:/localhost/scrsamples
